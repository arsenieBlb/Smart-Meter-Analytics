# Dashboard Requirements

## Audience

The dashboard is for utility leadership, operations teams, and analytics stakeholders who need a reliable view of smart-meter performance.

## Functional Requirements

- Show consumption trends over time.
- Support filtering by date, region, meter type, and customer type.
- Show executive KPIs for consumption, active meters, reading success, missing readings, anomalies, and critical meters.
- Identify meters with poor health, high anomaly counts, stale data, or missing readings.
- Provide a metric definitions page for self-service interpretation.

## Data Requirements

- Use the PostgreSQL star schema or CSV exports generated by the Python pipeline.
- Use `dim_date` as the date table.
- Use DAX measures from `dax_measures.dax`.
- Use SQL validation checks to confirm data quality before screenshots are captured.

## Design Requirements

- Use consistent colors for meter health status:
  - Healthy: green
  - Watch: amber
  - Critical: red
  - Inactive: gray
- Keep slicers visible on analytical pages.
- Use clear titles and business-friendly labels.
- Prioritize KPI cards and trend visuals on the first page.

## Acceptance Criteria

- All five report pages are created.
- All required DAX measures are added and formatted.
- Relationships match `semantic_model_guide.md`.
- KPI cards respond to slicers.
- Critical meter table can be sorted by health score.
- Screenshots are added to the GitHub README or documentation folder after dashboard creation.
